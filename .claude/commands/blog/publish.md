---
description: ブログ記事を公開可能な状態に整える
argument-hint: [記事のパス。省略時は最新の draft 記事]
---

`.claude/skills/blog-writing/SKILL.md` の「公開準備チェックリスト」を実行して、記事を `draft: true` から外せる状態にする。

対象: $ARGUMENTS

手順:

1. SKILL.md を読み込む
2. 対象ファイルが指定されていなければ `src/posts/*.md` のうち `draft: true` を含むファイルを列挙する。1 本だけならそれを対象に、複数あれば AskUserQuestion でユーザーに選んでもらう
3. SKILL.md の **公開準備チェックリスト** を実行する:
   - スクショ placeholder (`<!-- スクショ: ... -->` と `> 📸 ...`) の残存を Grep で検知する。残っていれば「画像差し替え」「全削除」「draft のまま中断」のどれかを AskUserQuestion で確認する
   - 「画像差し替え」が選ばれた場合は SKILL.md の「画像ファイルの管理」に従う:
     - `static/posts/<slug>/` ディレクトリの存在を確認 (無ければ作成を提案)
     - ディレクトリ内の画像ファイル一覧を `ls static/posts/<slug>/` で取得し、placeholder の番号 (① ② ...) と `NN-` 連番をマッピング
     - placeholder の 2 行ペアを `![alt](/posts/<slug>/NN-...png)` 1 行に置換 (alt は placeholder 内の説明文を流用)
     - 画像ファイルが揃っていない場合はユーザーへ不足分を伝えて中断
   - 「全削除」が選ばれた場合は `<!-- スクショ: ... -->` と `> 📸 ...` の 2 行ペアを丸ごと削除 (前後の空行も整える)
   - `## まとめ` と `## 参照` が末尾にあることを確認 (無ければ追記を提案)
   - `date` が意図した日付か AskUserQuestion で確認 (今日に更新するか、初稿日のまま残すか)
4. 上記が解決したら `draft: true` の行を削除する
5. `pnpm lint` を実行し、新規違反が無いことを確認。違反があれば修正してから再実行
6. 必要に応じて `pnpm check` を実行
7. `git status` を出して、ステージング状態をユーザーに見せる
8. **コミットや push は自動でやらない**。最終確認はユーザーが行う
