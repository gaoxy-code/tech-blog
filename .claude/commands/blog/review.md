---
description: ブログ記事の構成・規約を添削する
argument-hint: [記事のパス。省略時はステージング中または最新更新の記事]
---

`.claude/skills/blog-writing/SKILL.md` の「添削チェックリスト」を実行して、構成と規約のズレを指摘する。

対象: $ARGUMENTS

手順:

1. SKILL.md を読み込む
2. 対象ファイルが指定されていなければ次の優先順位で 1 本に決める:
   - `git status` で `src/posts/*.md` のステージング済みファイル
   - 上記が無ければ `git status` で `src/posts/*.md` の未ステージング変更
   - 上記が無ければ `src/posts/*.md` のうち `date` が最新のもの
   - どれも該当しなければ「対象ファイルを引数で指定してください」と返してユーザーに尋ねる
3. SKILL.md の **添削チェックリスト** を 1 項目ずつ評価する。違反があれば「行番号 + 何が違反か + 修正案」を提示
4. 違反が機械的に直せるもの (フロントマターのインデント、`## 参考リンク` → `## 参照` 等) は Edit で実際に修正する。スクショの扱いや draft の判断はユーザーへ確認
5. 文章の質や事実関係には踏み込まない (それは `/blog:proofread` の責務)
6. 最後に「適用した修正」「未対応で確認が必要な事項」「クリアできた項目」を簡潔にサマリ
