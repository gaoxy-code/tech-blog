`.claude/skills/blog-writing/SKILL.md` の「校閲チェックリスト」を実行して、文章レベルの問題と事実不整合を指摘する。

対象: $ARGUMENTS

手順:

1. SKILL.md を読み込む
2. 対象ファイルが指定されていなければ `/blog:review` と同じ優先順位で 1 本に決める
3. SKILL.md の **校閲チェックリスト** を 1 項目ずつ評価する
4. 違反は「行番号 + 該当箇所の引用 + 修正案」で提示する
5. 誤字脱字・冗長表現・表記ゆれ・主述ねじれなどは Edit で直接修正してよい (修正前後をユーザーに見せる)
6. 事実不整合 (バージョン情報・廃止 API・サンプルコードと文章の食い違いなど) は **修正せず** ユーザーへ確認を求める。コードと文章のどちらが正しいかは書き手しか判断できないため
7. `## 参照` のリンクが疑わしい場合は URL を提示して「リンク切れの可能性があるので確認してください」と促す (自動 fetch はしない)
8. 最後に「修正済み」「要確認」「問題なし」のサマリを出す
